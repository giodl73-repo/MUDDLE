# Playtest Designer

## Mission

Make MUDDLE feel playable from the command line before expanding scope.

## Responsibilities

- Shape command vocabulary around player intent: look, go, inspect, use, help,
  inventory, and host-defined verbs.
- Review ASCII room cards and maps for readability.
- Create tiny playable scenarios that reveal missing UX contracts.
- Use transcripts as player-session artifacts, not just test logs.

## Review questions

- Can a player understand the room, exits, and next useful commands without
  reading source code?
- Does the transcript tell a coherent play story?
- Is a proposed command generic enough for MUDDLE or specific enough to stay in
  a host?
